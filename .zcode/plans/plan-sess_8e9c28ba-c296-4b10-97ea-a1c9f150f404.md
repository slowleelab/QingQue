# 工程结构整理方案（止血 + 归位 + 去重）

目标：让 `main` 恢复到"clone 下来能跑"的开源就绪状态，清除历史污染，收敛散落配置。

## 执行顺序（每步可独立验证）

### 第 0 步 · 安全前置
- 先 `git stash list` / `git status` 快照，确认无遗漏。
- **不会**触碰任何业务代码逻辑，只做"入库、移动、去重、清污"。

### 第 1 步 · 修正 .gitignore（堵住污染源头）
在 `.gitignore` 补充：
```
# 前端产物（agent 是 Python 项目，这些属误装/缓存）
node_modules/
.vite/
dist/
test-results/
package-lock.json   # 若确认 agent 不需要 JS 工具链
```
- 注意：`web/` 下的 `dist/`、`node_modules/` 已被忽略，无需重复；新增规则用全局 pattern 覆盖所有子项目。

### 第 2 步 · 清除已入库的构建产物
- `git rm -r --cached agent/.vite`（从历史摘除，保留磁盘文件无意义，直接删）
- 删除误装的 JS 工具链残留：`agent/package.json`、`agent/package-lock.json`、`agent/node_modules/`（已确认仅含 Playwright 且无任何 spec/config 引用）
- 这些进"清理"动作，不混入功能 commit。

### 第 3 步 · 整理工作区改动并验证（单一大 commit）
按你选的策略：
1. **先跑验证**：`cd agent && poetry run ruff check smartcs/ tests/ && poetry run ruff format` + `make test`
   - 若 lint/test 报错，**先报告给你**，不擅自改业务代码；由你决定是修还是先入库。
2. 验证通过后，将 154 个改动（71 修改 + 26 删除 + 57 新增，含 PII/审计/限流/安全/OE管道/FAQ路由/bot_agent/temporal + 7 个 migration + 对应测试）`git add -A` 一次性提交。
   - commit message 建议：`feat: Sprint5 功能集 — 安全/审计/OE管道/坐席辅助增强`（可改）
   - **删除的 26 个文件**（ai_executor_dag、state_manager、bot/agent.py、workflows/ 等）一并随该 commit 落库。

### 第 4 步 · knowledge-platform 并入本仓库
- `git add knowledge-platform/`（Python 微服务，结构规范），纳入 monorepo。
- 在其 README 顶部补一行"本服务与 SmartCS 的关系"说明（可选，属文档完善）。
- 单独一个 commit：`feat: 并入 knowledge-platform 知识数据微服务`。

### 第 5 步 · 配置归位（消除三处散落）
- **删除重复**：`agent/config/prometheus.yml` 与 `config/prometheus.yml` 内容完全相同 → 保留一处（建议统一放 `config/`，供 deploy 引用），删另一份。
- **Grafana 三处**（`config/grafana`、`agent/config/grafana`、`deploy/grafana`）：确认为唯一事实源（建议 `config/grafana`），其余改为引用或删除。
- ⚠️ 改配置路径会影响 `deploy/docker-compose.yml` 的挂载路径——我会先 grep 所有引用再动，动完报告改了哪些引用。

### 第 6 步 · 文档去重 + 图片归位
- `CLAUDE.md` 与 `AGENTS.md` 逐字符重复 → 保留 `AGENTS.md` 为事实源，`CLAUDE.md` 改为**软链**（`ln -sf AGENTS.md CLAUDE.md`），工具两边都能读。
- 散落图片归位：`star-connection/*.png`、`star-connection/test/*.png` → 移到 `docs/assets/` 或 `star-connection/docs/assets/`，并更新引用它们的 md。
- ⚠️ 中文文件名（git 显示为转义路径）：是否保留中文名？我倾向保留（可读性好），但若你要跨平台协作可改英文——**这点请告诉我**。
