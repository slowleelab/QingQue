混合检索 (BM25 + Vector + RRF)
1. 查询嵌入 → Milvus 向量检索 (top_k×3)
2. 查询文本 → ES BM25 检索 (top_k×3)
3. RRF 融合: score = Σ(1/(k + rank_i)), k=60
4. 可选 Reranker 精排
5. 置信度阈值过滤 → 截断 top_k
降级矩阵
ES ✅ Milvus ✅ → 混合检索 (最优)
ES ✅ Milvus ❌ → BM25 Only
ES ❌ Milvus ✅ → 向量 Only
ES ❌ Milvus ❌ → 空结果
Embedding 熔断 → 自动降级 BM25

五、关键设计决策
#决策选择原因
1阶段模型BOT / AGENT / ENDED 三阶段 + 7 子状态HANDOFF 不是独立阶段（排队是人工服务的冷启动）；ASSIST 是技术能力不是业务阶段；ENDED 子状态是元数据不是状态
2客户界面单一聊天框，不切换客户视角：始终在一个聊天框对话，无论对面是机器人还是坐席
3排队与振铃分离AG_QUEUED / AG_ASSIGNED 分开行业标准（Amazon Connect/Genesys/Twilio）：系统行为不同、超时含义不同、SLA 指标不同
4异步消息队列Redis List + BRPOP轻量, 避免阻塞 HTTP, 共享 Redis
5双通道分类Rule 快路 + LLM 慢路平衡速度与精度
6OE 状态机Temporal Workflow + 同步降级可观测+可恢复+策略矩阵
7营销压制D1 激活 → D2 suppress 2 轮避免服务场景过度营销
8反馈缓冲3 秒延迟提交 + Undo坐席误操作可撤销
