# SmartCS 智能客服平台 UAT 测试案例

> **版本**: v2.1
> **日期**: 2026-05-25
> **测试类型**: 用户验收测试（User Acceptance Testing）
> **测试方式**: 浏览器自动化 (Playwright) 或人工操作
> **前端技术栈**: Vue 3 + Element Plus（客户页 + 坐席页统一）
> **测试范围**: 客户自助咨询 → 转人工 → 坐席辅助 → 话后小结 完整业务闭环

---

## 一、角色与操作约定

| 角色 | 访问地址 | 界面说明 |
|------|---------|---------|
| **客户 (CU)** | `http://localhost:8080/` | Vue 客户聊天页：登录 → 发送消息 → 查看 Bot/坐席回复 |
| **坐席 (AG)** | `http://localhost:8080/agent` | Vue 坐席工作台：会话列表 → 辅助面板 → WS 推送 |
| **运营 (OP)** | 管理后台 | 上传文档，管理知识库 |

**前端入口统一由 Nginx (port 8080) 提供服务，API 代理到 Bot (:8000) / Assist (:8001)。**

## 二、端到端业务场景

### 场景 A：客户自助咨询 → 满意解决

```
前置条件: Bot 服务运行中，知识库已导入

1. 客户打开 http://localhost:8080/
   → 页面显示登录表单（客户ID + 名称）

2. 客户填写信息，点击"开始咨询"
   → 进入聊天界面，显示欢迎消息
   → 侧边栏显示"会话信息"（会话ID、状态、消息数）

3. 客户输入"你好"，点击发送
   → 消息气泡显示在右侧（蓝色）
   → 输入框自动清空
   → 等待 ~2-5s，Bot 回复显示在左侧

4. 客户输入"我上个月账单多少钱"
   → Bot 回复账单查询指引

5. 客户输入"好的谢谢"
   → Bot 回复结束语

验证点:
  - 登录后可正常发送消息
  - Bot 回复在 5s 内到达
  - 消息按顺序显示，客户右侧，Bot 左侧
  - 侧边栏消息数实时更新
  - 全程未触发转人工
```

### 场景 B：客户转人工 → 坐席辅助推送

```
前置条件: star-conn 运行中，至少 1 名坐席在线

1. 客户输入"我要转人工"
   → 系统消息显示"正在转接人工客服..."
   → 侧边栏状态变为"排队等待"

2. 坐席打开 http://localhost:8080/agent
   → 页面显示会话列表（左侧）+ AI 辅助面板（右侧）
   → 会话列表中出现新会话

3. 坐席点击会话
   → 右侧展开对话面板 + 辅助推送面板
   → 辅助面板显示推荐话术 / 知识片段

4. 客户继续输入"账单怎么查"
   → 坐席端辅助面板刷新新的推送内容

验证点:
  - 客户触发转人工后看到转接提示
  - 坐席端会话列表可见新会话
  - 坐席点击会话后辅助面板出现
  - 推送内容与客户问题相关
```

### 场景 C：合规风险告警

```
前置条件: AlertEngine 已加载规则

1. 客户发送含"保证"的消息
   → 坐席端辅助面板弹出风险徽章
   → 营销卡片缩小或隐藏
```

### 场景 D：坐席保持与恢复

```
1. 坐席点击"保持"按钮 → 状态切换为 on_hold
2. 60s 无客户消息 → 坐席端弹出静音提醒
3. 坐席点击"恢复" → 状态恢复 active
```

### 场景 E：话后小结

```
1. 坐席点击"生成小结" → LLM 自动生成小结
2. 坐席审核/修改小结
3. 坐席点击"提交" → 会话结束
```

## 三、自动化测试覆盖

Playwright 测试脚本位于 `web/tests/uat/`。

### 客户聊天页（5 条）

| 测试 | 验证点 | 结果 |
|------|--------|------|
| 登录后显示欢迎 + 侧边栏 | 消息气泡、会话信息、机器人服务状态 | ✅ |
| 发送消息后收到回复 | textarea 清空、Bot 回复出现 | ✅ |
| 转人工触发 | 系统消息显示转接提示 | ✅ |
| 空消息拦截 | 纯空格输入时发送按钮 disabled | ✅ |

### 坐席工作台（4 条）

| 测试 | 验证点 | 结果 |
|------|--------|------|
| 会话列表 + 空状态 | session-list 可见、空状态提示 | ✅ |
| 点击会话展示辅助面板 | assist-panel 可见 | ✅ |
| 会话项 phase 标签 | 标签值为 机器人/坐席辅助/已结束 | ✅ |
| 跨窗口联动 | 客户发送后坐席端正常加载 | ✅ |

**运行方式**:
```bash
cd web && pnpm exec playwright test tests/uat/ --reporter=list
```

## 四、Playwright 测试执行指南

### 安装

```bash
cd web
pnpm add -D @playwright/test
pnpm exec playwright install chromium
```

### 运行

```bash
# 全部测试
pnpm exec playwright test tests/uat/

# 单场景
pnpm exec playwright test tests/uat/scenario_customer_chat.spec.ts

# UI 模式
pnpm exec playwright test --ui
```
